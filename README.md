# Building GenAI Applications on Databricks -- REPOS

Most AI courses stop at the model. 
This REPOS starts where the real work begins — turning a capable language model into a reliable, auditable, production-grade system that your organization can actually trust.

Across six tightly sequenced chapters, you will build the full stack of a modern GenAI application: from your first traced LLM call all the way through to an autonomous agen, with governance, evaluation, and monitoring baked in at every layer.

Every concept is immediately reinforced by a hands-on notebook on the Databricks platform that are versionned on this REPOS. 

## What you need coming in

-  Comfort with Pyspark/python programming
-  Basic understanding of ML concepts
-  Familiarity with SQL and data pipelines
-  A GitHub account for version control
-  A Databricks Free Edition workspace (set up in chapter 1)

## Chapter One : Foundations & Environment Setup
Discover the Databricks Free Edition, connect GitHub for version control, master prompt engineering, and deploy your first Claude Sonnet endpoint — all traced end-to-end in MLflow. 

## Chapter Two : Safety, Compliance & Access Control
Build the governance backbone for production AI: Unity Catalog namespacing and lineage, dedicated service principals, AI Gateway guardrails, rate limits, and inference tables. 

## Chapter Three : Retrieval-Augmented Generation
Turn unstructured enterprise data (PDFs, emails, scans) into a queryable knowledge base. 
Build a full hybrid pipeline: 
- chunking, 
- embeddings, 
- vector search, 
- keyword search, 
- and reranking. 

## Chapter Four : Autonomous Agentic Systems
Go beyond fixed chains. 
Build agents that plan, call tools, and adapt — using LangGraph and the Mosaic AI Agent Framework. 
Cover Genie, Unity Catalog functions, and MCP integrations. 

## Chapter Five : Quality Measurement & Improvement
Infrastructure metrics don't catch wrong answers. 
Build rigorous evaluation datasets, apply code-based scorers and LLM-as-judge, compare versions side-by-side, and close the improve loop. 

## Chapter Six : LLMOps & Production Automation
Ship your agent to production with no manual steps. 
Automated Databricks Jobs, Lakeflow pipelines from source to Gold, SQL dashboards, drift alerts, and a fully automated model deployment pipeline. 

## What you will build

- A working Claude Sonnet endpoint with full MLflow observability
- A governed data infrastructure secured by Unity Catalog and service principals
- A hybrid RAG pipeline over your own unstructured document corpus
- A production-ready LangGraph agent with tool use and multi-agent orchestration
- A rigorous evaluation framework with custom LLM judges and version comparison
